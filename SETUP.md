# Slava — что подкинуть, чтобы сайт ожил

## 1. Твоя фотография для Hero

Положи файл в `public/`:

- `public/me.png` ← **лучший вариант:** PNG с прозрачным фоном (вырезка)
- или `me.webp` / `me.jpg` / `me.jpeg`

С PNG-вырезкой эффект «Слава за гигантским именем» сыграет в полную силу. Снимаешь портрет на однотонном фоне → `remove.bg` (бесплатно, 60 сек) → кладёшь файл сюда.

Пока файла нет — рендерится плейсхолдер из `me-placeholder.jpg`, его удалять не нужно, он сработает только если других вариантов нет.

## 2. Telegram реле-бот

**Архитектура:** клиент пишет боту → бот пересылает тебе → ты отвечаешь Reply'ем → бот шлёт ответ клиенту. Клиент никогда не видит твой личный аккаунт.

### Что уже сделано

- Бот создан: `@gallogramer_bot`
- Токен в `.env.local`
- Код реле-бота: `app/lib/relay.ts`
- Webhook для прода: `app/api/telegram-webhook/route.ts`
- Локальный polling-режим для тестов: `scripts/bot-dev.mjs`

### Шаги для тебя — узнать свой chat_id и протестировать

1. **Запусти бота локально** в отдельном терминале:
   ```bash
   cd gallogramer-site
   npm run bot
   ```
   Увидишь `🤖 gallogramer bot — polling mode`.

2. **В Telegram:** найди `@gallogramer_bot` и напиши `/start`. Бот ответит твоим `chat_id` (например, `123456789`).

3. **Вставь chat_id в `.env.local`:**
   ```
   TELEGRAM_BOT_TOKEN=8519120598:...
   TELEGRAM_CHAT_ID=123456789       ← твой реальный ID
   ```
   `8519120598` оставить НЕЛЬЗЯ — это ID самого бота, бот не может писать сам себе.

4. **Перезапусти `npm run bot`** (Ctrl+C + снова) — теперь он знает админа.

5. **Проверь реле:**
   - Попроси кого-то ещё (или вторым аккаунтом) написать `@gallogramer_bot`
   - Сообщение прилетит к тебе с заголовком `📥 Новое сообщение клиенту`
   - **Сделай Reply** на это сообщение → твой ответ уйдёт клиенту

### Команды админа в боте

- `/help` — справка
- `/id` — показать твой chat_id

### Прод (Vercel)

После деплоя на Vercel:
1. В Vercel → Settings → Environment Variables добавить `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID`
2. Один раз вызвать (замени `<DOMAIN>` на свой):
   ```
   https://api.telegram.org/bot<TOKEN>/setWebhook?url=https://<DOMAIN>/api/telegram-webhook
   ```
3. Polling-скрипт больше не нужен — webhook возьмёт всё на себя.

## 3. Форма «Связаться» на сайте

Использует тот же бот: при отправке формы клиент НЕ получает чат, а тебе в бот приходит заявка с пометкой `📩 Заявка с сайта` (имя/контакт/сообщение). Дальше связываешься с клиентом тем способом, который он указал.

Если хочешь — форму можно вообще убрать, оставив только кнопку «Telegram bot» (она уже в footer).

## 4. Pinterest — синхронизация портфолио

После настройки: запускаешь `npm run sync-pinterest`, скрипт тянет все твои пины с Pinterest, скачивает картинки в `public/pinterest/`, обновляет манифест `app/data/photos.ts`. Сайт сразу показывает свежее портфолио.

### Шаги для тебя — получить access token

1. **Зайди на `https://developers.pinterest.com`** под своим Pinterest-аккаунтом.

2. **«Connect app»** (вверху). Заполни:
   - **App name:** `gallogramer-site` (или любое имя)
   - **App description:** `Personal portfolio sync`
   - **Website:** `https://gallogramer.com`
   - **Category:** Productivity / Personal use (что предложат)

3. **Подтверди создание.** Pinterest выдаст App ID и App Secret — пока их не трогаем.

4. **Перейди в свой app → раздел «Access Token» (или «OAuth»).** Найди кнопку **«Generate access token»** или **«Add new trial access token»**.

5. **Выбери scopes:**
   - `boards:read`
   - `pins:read`

6. **Сгенерируй токен.** Он будет длинной строкой типа `pina_AMA...`. Скопируй.

7. **Открой `.env.local`** и добавь строку:
   ```
   PINTEREST_ACCESS_TOKEN=pina_AMA...
   ```

8. **Запусти `npm run sync-pinterest`.** Скрипт сначала покажет список твоих досок:
   ```
   Найдено досок: 3
     · Portfolio — 47 пинов  (id: 9123456789)
     · Inspiration — 12 пинов  (id: 9987654321)
   ```
   Затем начнёт качать все пины.

### Из конкретной доски

Если хочешь синхронизировать только одну доску (например, рабочее портфолио), скопируй её ID из вывода выше и добавь в `.env.local`:
```
PINTEREST_BOARD_ID=9123456789
```
Запусти sync заново — скрипт возьмёт только эту доску.

### Регулярная синхронизация

Пока — запускаешь `npm run sync-pinterest` руками когда добавил новые пины. Это даёт полный контроль и нулевую стоимость.

Когда переедем на Vercel, автоматизируем через **Vercel Cron** — раз в час/день фоновая функция вызывает тот же код. Об этом поговорим перед деплоем.

### Срок жизни токена

Trial access tokens живут **30 дней**. Через 30 дней — перегенерируй в дашборде Pinterest и обнови `.env.local`. Для долгосрочного токена нужен полный OAuth-flow (refresh tokens) — это можно сделать на этапе деплоя.

### Что если что-то не так

- **`401 Unauthorized`** — токен невалидный или истёк. Перегенерируй.
- **`403 Forbidden`** — у токена нет нужных scopes. Перегенерируй с `boards:read,pins:read`.
- **Пустой список пинов** — проверь что в Pinterest-аккаунте действительно есть пины. Если все пины в досках — попробуй без `PINTEREST_BOARD_ID`.
